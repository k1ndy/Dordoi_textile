# Dordoy Textile — каталог одежды с рынка «Дордой»

Современный сайт-каталог одежды с рынка **Дордой** для продаж по СНГ: розница, опт,
заявки от селлеров и отшив крупных партий под бренд. Есть админ-панель, формы заявок,
Telegram-уведомления, мультиязычность и мультивалютность.

> **Сайт запускается сразу** — даже без базы данных он работает на демо-данных.
> Для реального использования подключите Supabase (см. ниже).

## ✨ Возможности

- **Главная** с преимуществами, категориями, новинками и блоком для селлеров.
- **Каталог** с фильтрами: категория, пол, размер, цвет, сезон, страна, цена, наличие, новинки, опт/розница.
- **Карточка товара**: галерея, цены (розница/опт), размеры, цвета, характеристики, форма заказа, блок отшива под бренд.
- **Страницы для бизнеса**: Оптовикам, Для селлеров, Отшив крупной партии, Доставка и оплата, О нас, Контакты.
- **3 типа заявок** (розница / опт / селлеры) с валидацией и защитой от спама (honeypot + rate-limit).
- **Админ-панель** (`/admin`): дашборд, товары (CRUD, фото, новинки/хиты, наличие, SEO), категории (порядок, видимость), заявки (статусы, детали), настройки сайта.
- **Telegram-уведомления** о новых заявках владельцу.
- **SEO**: человекопонятные URL, метатеги для категорий и товаров, `sitemap.xml`, `robots.txt`, Schema.org.
- **Мультиязычность** (RU основной; структура готова к kg/kz/uz/en) и **мультивалютность** (сом/₽/₸/$).
- **Адаптивный дизайн** — телефон, планшет, ПК.

## 🛠 Стек

- **Next.js 14** (App Router) + **TypeScript**
- **Tailwind CSS** (кастомная тема, шрифты Unbounded + Golos Text)
- **Supabase** — БД, авторизация админа, хранилище фото
- Деплой: **Vercel** / **Netlify**

## 🚀 Быстрый старт

```bash
npm install
npm run dev
```

Откройте http://localhost:3000 — сайт работает на демо-данных.
Админ-панель: http://localhost:3000/admin (демо-вход: `admin` / `admin123`).

## 🔌 Подключение Supabase (для реальной работы)

1. Создайте проект на [supabase.com](https://supabase.com).
2. Выполните `supabase/schema.sql` в **SQL Editor** (создаст таблицы, RLS, бакет, демо-данные).
3. Создайте администратора: **Authentication → Users → Add user** (email + пароль).
4. Скопируйте `.env.example` в `.env.local` и заполните:

```env
NEXT_PUBLIC_SUPABASE_URL=...
NEXT_PUBLIC_SUPABASE_ANON_KEY=...
SUPABASE_SERVICE_ROLE_KEY=...        # только сервер, держать в секрете
```

5. Перезапустите `npm run dev`. Теперь вход в `/admin` — по email/паролю,
   а товары/категории/заявки/настройки сохраняются в БД.

## 📨 Telegram-уведомления

1. Создайте бота через [@BotFather](https://t.me/BotFather) → получите токен.
2. Узнайте свой `chat_id` (напишите боту, откройте
   `https://api.telegram.org/bot<TOKEN>/getUpdates`).
3. В `.env.local`:

```env
TELEGRAM_BOT_TOKEN=...
TELEGRAM_CHAT_ID=...
```

Теперь каждая новая заявка приходит вам в Telegram.

## 🔐 Безопасность

- Админ-маршруты защищены `middleware.ts` (Supabase-сессия или демо-cookie).
- Запись заявок — через серверный API с **service role** (ключ не попадает на клиент).
- **RLS** в Supabase: публично читаются только видимые товары/категории; заявки недоступны публично.
- Формы: валидация, honeypot, ограничение частоты отправки.
- Удаление товаров/категорий — с подтверждением.
- Загрузка фото — в защищённый бакет (запись только для авторизованных).

## 📁 Структура

```
src/
  app/
    (site)/            публичные страницы (Header/Footer/контакты)
      page.tsx         главная
      catalog/         каталог + /[category]
      product/[slug]/  страница товара
      wholesale, sellers, manufacturing, delivery, about, contacts
    admin/             админ-панель (+ login)
    api/leads          приём заявок + Telegram
    api/admin/login    демо-вход
    sitemap.ts, robots.ts
  components/           UI и админ-компоненты
  lib/                 данные, типы, supabase, i18n, валюты
supabase/schema.sql    схема БД, RLS, бакет, сид
```

## ☁️ Деплой на Vercel

1. Запушьте репозиторий на GitHub.
2. Импортируйте в Vercel, добавьте переменные окружения из `.env.local`.
3. Deploy. `NEXT_PUBLIC_SITE_URL` укажите на боевой домен (для sitemap/SEO).
